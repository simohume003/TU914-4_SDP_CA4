# TU914-4_SDP_CA4
Repo for CA4 (Software Design Patterns)
This task will involve the creation of a Web application, using a framework and programming language or languages that you are familiar with, to interact with an underlying database storing information about the catalogue of an online clothes store. As part of the application, you should identify appropriate software patterns for tasks within the application, apply these patterns, and maintain both a repository of your progress using Github or some other version control system and produce a report describing the overall structure of your system and the situations where patterns were used.
The application should allow multiple simultaneous connections to an underlying database as well as the standard CRUD functionality for the application. 
Details held about stock items should include title, manufacturer, price, category, an associated image, and any other details required for the purpose of the application.
Customers should be able to create accounts, enter in suitable data for those accounts such as shipping address, payment method, etc., as well perform searches of the underlying database for items by category, by manufacturer or by title (with partial matching allowed for category and title). The result sets should be displayed by the application and the user should be able to sort the results interactively in ascending or descending order on the value of various attributes (title, manufacturer, price, etc.). The customer should be able to select items to purchase and add to a shopping cart, with the purchase being processed by the system and stock levels being updated to reflect the results of the purchase. Features such as discounting and loyalty cards for customers may be applied in specific circumstances as decided by the developer. Customers should be able to interactively rate the item on a scale of 1-5 and to leave comments/reviews of an item.
Administrators should be able to view stock items, search and update the stock database, displaying and sorting result sets in the same manner as customers can. They should be able to view customer details and their purchase histories. They should be able to simulate the purchase of stock items from wholesalers by adjusting stock levels.
Normal security measures (logins, passwords, etc.) should be enforced for customers and administrators. Appropriate test suites may be used.
Your task should include the design and implementation of an appropriate database structure as well as the construction of the application itself.
